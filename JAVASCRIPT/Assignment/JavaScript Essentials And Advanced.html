● What will be the result for these expressions?

1. 5 > 4

The expression 5 > 4 evaluates to true. This is because 5 is indeed greater than 4, and in JavaScript, when you use the > operator, it returns true if the left operand is greater than the right operand, otherwise, it returns false. So, in this case, the result will be true.

2. "apple" > "pineapple"

In JavaScript, when you compare strings using the > operator, it compares them based on their lexicographical (dictionary) order.
In this case, "apple" comes before "pineapple" in the dictionary, so "apple" > "pineapple" evaluates to false. This might seem counterintuitive at first, but it's because JavaScript compares the strings character by character, and in this comparison, the first characters "a" in both strings are the same, but the second character "p" in "apple" comes before the second character "i" in "pineapple", hence "apple" is considered less than "pineapple". Therefore, the result will be false.

3. "2" > "12"

In JavaScript, when comparing strings, the comparison is done character by character based on Unicode values.
So, in the expression "2" > "12", the character "2" has a higher Unicode value than "1", so the string "2" is actually greater than the string "12".
Therefore, "2" > "12" evaluates to true.

4. undefined == null

In JavaScript, undefined and null are two distinct values, but they are considered loosely equal (==) to each other.
So, undefined == null evaluates to true. However, it's important to note that === is the strict equality operator in JavaScript, and it would return false in this case because it compares both value and type, and undefined and null have different types.

5. undefined === null

In JavaScript, the === operator checks for strict equality, meaning it compares both the value and the type of the operands.
In the expression undefined === null, undefined is a type of undefined, while null is a distinct type representing an absence of value or "null". Therefore, undefined and null are not strictly equal.
So, undefined === null evaluates to false.

6. null == "\n0\n"

In JavaScript, the == operator performs type coercion, meaning it tries to convert operands to a common type before comparison.
In this case, null is a special value representing the absence of a value, while "\n0\n" is a non-empty string.
When comparing a non-empty string to null, JavaScript coerces null to convert it to a number, which becomes 0. Then, "\n0\n" is not an empty string, so it's coerced to a number, which becomes 0 due to the leading and trailing newline characters being ignored.
So, null == "\n0\n" evaluates to true because both are coerced to the number 0 during comparison.

7. 7. null === +"\n0\n"

In JavaScript, the === operator performs strict equality comparison, meaning it checks both the value and the type of the operands without type coercion.
In this expression, null is of type object, while +"\n0\n" is a string that's coerced into a number. The + operator in front of "\n0\n" coerces it into a number, which becomes 0.
Therefore, null === +"\n0\n" compares the null object to the number 0, and since they are of different types (object vs number), the result will be false.

● What is JSON

JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate. It is based on a subset of the JavaScript programming language, but is language-independent, meaning it can be used with most programming languages.

JSON is commonly used for transmitting data between a server and a web application, as well as for storing configuration settings and data in files. It is often used in web development for APIs (Application Programming Interfaces) where data needs to be exchanged between a client and a server.

JSON data is organized in key-value pairs, similar to JavaScript objects. It supports various data types, including strings, numbers, booleans, arrays, and objects. Here is an example of JSON data: 

            {
                "name": "John",
                "age": 30,
                "isStudent": false,
                "skills": ["JavaScript", "HTML", "CSS"],
                "address": {
                  "street": "123 Main St",
                  "city": "New York",
                  "country": "USA"
                }
              }

● What is promises

In JavaScript, a Promise is an object representing the eventual completion or failure of an asynchronous operation, and its resulting value. It allows you to handle asynchronous operations more easily and cleanly, avoiding deeply nested callback functions (known as "callback hell").

Promises have three states:

1. Pending: Initial state, neither fulfilled nor rejected.
2. Fulfilled: The operation completed successfully.
3. Rejected: The operation failed.
Promises can be created using the Promise constructor. The constructor takes a single argument, a function called the "executor" function, which takes two parameters: resolve and reject. Inside the executor function, you perform the asynchronous operation, and then call resolve(value) when it's successful or reject(reason) when it fails.

● Write a program of promises and handle that promises also


            const myPromise = new Promise((resolve, reject) => {
                // Perform an asynchronous operation
                setTimeout(() => {
                  const randomNumber = Math.random();
                  if (randomNumber > 0.5) {
                    resolve(randomNumber); // Resolve with a value
                  } else {
                    reject(new Error('Random number is too low')); // Reject with an error
                  }
                }, 1000);
              });
              
              // Consuming the Promise
              myPromise.then((value) => {
                console.log('Promise fulfilled with value:', value);
              }).catch((error) => {
                console.error('Promise rejected with error:', error);
              });
